# Demo 6：视频知识卡片

> 看视频，学知识，一键沉淀。面向抖音用户，把刷到的碎片知识变成可保存、可复用的结构化卡片。

## 背景与目标

用户在抖音刷到干货视频（美食教程、景点介绍、理财知识、职场技巧），当下觉得有用，但看完就忘，收藏了也不会再看。

**核心价值**：把视频里的知识自动提取出来，生成一张结构化卡片，用户可以保存、搜索、分享，把"刷视频"变成"真学习"。

## 用户场景

| 场景 | 输入 | 输出 |
|------|------|------|
| 看完美食教程 | 视频链接 | 一张带步骤和食材清单的菜谱卡片 |
| 看完旅游攻略 | 视频链接 | 一张带景点和路线的旅行卡片 |
| 看完职场技巧 | 视频链接 | 一张带要点和适用场景的工作卡片 |
| 看完知识科普 | 视频链接 | 一张带核心知识点的小卡 |

## 设计原则

- **一键操作**：用户只需要粘贴一个视频链接
- **知识结构化**：不是简单摘要，而是提取可保存的结构化信息
- **可编辑**：生成后用户可微调内容
- **可导出**：支持复制、分享到微信/微博

## 功能范围

### 核心功能（MVP）

1. **视频链接输入**
   - 支持抖音/B站/YouTube 视频链接
   - 输入后自动解析，不需要用户操作

2. **视频内容理解**
   - 提取视频的核心知识点（非全文摘要）
   - 根据视频类型（美食/旅行/职场/科普）自动适配提取模板
   - 保留关键细节（数字、步骤、注意事项）

3. **知识卡片生成**
   - 根据内容类型生成对应结构的卡片
   - 卡片包含：标题、核心知识点、补充信息、操作建议
   - 支持一键复制全文

4. **多类型支持**

   **美食类卡片结构：**
   ```json
   {
     "type": "food",
     "title": "菜谱名称",
     "difficulty": "简单/中等/困难",
     "time": "烹饪时长",
     "ingredients": ["食材1", "食材2"],
     "steps": ["步骤1", "步骤2"],
     "tips": ["小贴士1", "小贴士2"],
     "applicable": "适合人群"
   }
   ```

   **旅行类卡片结构：**
   ```json
   {
     "type": "travel",
     "title": "目的地",
     "highlights": ["必去景点1", "必去景点2"],
     "route": "推荐路线",
     "budget": "预算参考",
     "bestSeason": "最佳季节",
     "tips": ["注意事项1", "注意事项2"]
   }
   ```

   **职场类卡片结构：**
   ```json
   {
     "type": "career",
     "title": "技巧名称",
     "corePoint": "一句话核心观点",
     "keyPoints": ["要点1", "要点2", "要点3"],
     "applicable": "适用场景",
     "example": "应用举例"
   }
   ```

   **科普类卡片结构：**
   ```json
   {
     "type": "science",
     "title": "知识点名称",
     "oneLine": "一句话解释",
     "details": "详细说明",
     "relatedExamples": ["例子1", "例子2"],
     "tags": ["标签1", "标签2"]
   }
   ```

### 扩展功能（V2）

1. **视频类型自动识别**：根据视频标题/内容自动判断卡片类型
2. **卡片编辑**：用户可在生成后手动调整内容
3. **收藏夹**：历史生成的卡片支持搜索和回顾
4. **分享卡片**：生成图片格式的卡片，可分享到社交平台
5. **追问深化**：用户可对某个知识点追问，获取更详细的解释

## 技术方案

### 整体流程

```
视频链接
    │
    ├─→ Demo1 视频解析 → 关键帧 + 音频
    │
    ├─→ 视频类型识别（根据标题/标签/关键帧判断）
    │
    ├─→ 音频转写 → 结构化文本
    │
    └─→ 知识提取 → 根据类型应用对应模板 → 生成知识卡片
```

### 技术栈

| 组件 | 方案 | 说明 |
|------|------|------|
| 视频解析 | 复用 Demo1 | 复用已有视频解析能力 |
| 音频转写 | DashScope ASR | qwen3-asr-flash |
| 内容理解 | 通义千问 | qwen-plus，结构化信息提取 |
| 前端 | Vue 3 + Vite | 复用项目技术栈 |
| 后端 | Hono | 复用项目技术栈 |

### API 设计

**POST /api/card**

输入视频链接，生成知识卡片

Request:
```json
{
  "url": "视频链接"
}
```

Response:
```json
{
  "taskId": "xxx"
}
```

**GET /api/card/:taskId**

查询卡片生成结果

Response:
```json
{
  "id": "xxx",
  "status": "done",
  "card": {
    "type": "food",
    "title": "蒜香排骨",
    "difficulty": "中等",
    "time": "40分钟",
    "ingredients": ["排骨500g", "蒜末30g", "生抽2勺"],
    "steps": ["排骨焯水去腥", "热油煸炒至金黄", "加入蒜末和调料"],
    "tips": ["排骨要选小排", "蒜末要分两次放"],
    "applicable": "厨房新手及以上"
  }
}
```

## 与其他 Demo 的关系

- **依赖 Demo1**：视频解析（复用）
- **依赖 Demo3**：音频转写（参考）
- **是 Demo3 的下游**：Demo3 是通用摘要，本 Demo 是垂直场景的结构化卡片
- **与 Demo4 互补**：Demo4 生成营销内容，本 Demo 生成知识沉淀

## 注意事项

- 视频类型识别用关键词匹配（标题/标签），不需要复杂 ML
- 卡片模板纯本地处理，不需要额外 API 调用
- 知识提取的 prompt 需要针对每个类型单独优化
- 注意控制 token 消耗，长视频分段处理
