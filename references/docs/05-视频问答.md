# Demo 5：视频问答（Video QA）

> 展示"AI 看懂了"——两个赛道通用的交互形式。

## 目标

输入一段视频 + 用户的自然语言问题，返回基于视频内容的回答。
- "视频里在做什么菜？"
- "这家店在哪里？"
- "视频中出现了几个角色？"
- "这个产品有什么特点？"

## 技术栈

| 组件 | 方案 | 理由 |
|------|------|------|
| 视频预处理 | 复用 Demo 1 | 抽帧 + 音频转录 |
| 上下文构建 | 关键帧 + 转录文本拼装 | 把视频变成可问答的上下文 |
| 问答 | `@anthropic-ai/sdk`（多模态） | 支持图片+文本混合输入 |

## 核心流程

```
视频 + 用户问题
  │
  ├─→ Demo 1: 抽取关键帧 + 音频转文字
  ├─→ 构建上下文（关键帧图片 + 转录文本 + 时间戳）
  ├─→ 拼装 prompt（上下文 + 用户问题）
  └─→ 调用多模态 LLM → 返回答案
```

## 关键代码示例

```typescript
import Anthropic from '@anthropic-ai/sdk'
import fs from 'fs/promises'
import type { MessageParam } from '@anthropic-ai/sdk/resources/messages'

async function videoQa(
  frames: string[],
  transcript: string,
  question: string,
): Promise<string> {
  const content: MessageParam['content'] = []

  // 添加关键帧图片（限制帧数控制 token）
  for (const framePath of frames.slice(0, 8)) {
    const imgB64 = (await fs.readFile(framePath)).toString('base64')
    ;(content as any[]).push({
      type: 'image',
      source: {
        type: 'base64',
        media_type: 'image/jpeg',
        data: imgB64,
      },
    })
  }

  // 添加转录文本和问题
  ;(content as any[]).push({
    type: 'text',
    text: `以下是视频的关键帧和音频转录：

=== 音频转录 ===
${transcript}

=== 用户问题 ===
${question}

请基于视频内容回答问题。如果视频中没有相关信息，请说明。`,
  })

  const client = new Anthropic()
  const response = await client.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1024,
    messages: [{ role: 'user', content }],
  })

  return response.content[0].text
}
```

## 优化方向

- **帧数选择**：不是所有帧都需要，可以根据问题类型智能选帧
- **对话式**：支持追问，上下文对话
- **引用来源**：回答时标注"在第 X 秒的画面中"，增强可信度
- **Vue 界面**：左侧视频预览，右侧对话框
