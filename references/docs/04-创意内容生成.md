# Demo 4：场景/内容 → 创意内容生成

> 赛道一"按下快门之后，还能发生什么？"的典型场景。

## 目标

输入图片或视频理解结果，输出创意内容：
- 小红书/朋友圈/抖音文案
- 相关知识科普
- 商品推荐/找同款描述
- 旅行攻略/美食指南

## 技术栈

| 组件 | 方案 | 理由 |
|------|------|------|
| 内容生成 | `@anthropic-ai/sdk` | 指令遵循好，文风可控 |
| Prompt 模板 | TypeScript 模板字符串 | 类型安全，快速切换风格 |
| 备选 | 通义千问 API | 国内场景更自然 |

## 核心流程

```
图片/理解结果
  │
  ├─→ 选择生成模板（小红书/科普/推荐/攻略）
  ├─→ 拼装 prompt（模板 + 内容上下文 + 风格指令）
  └─→ 调用 LLM → 生成内容
```

## 关键代码示例

```typescript
import Anthropic from '@anthropic-ai/sdk'

type Style = 'xiaohongshu' | 'science' | 'recommend' | 'travel'

const TEMPLATES: Record<Style, (context: string) => string> = {
  xiaohongshu: (ctx) => `你是一个小红书博主，请根据以下内容写一篇小红书笔记。
要求：标题吸引眼球，用 emoji 装饰，口语化，有互动感，200字以内。

内容：${ctx}`,

  science: (ctx) => `你是一个科普作者，请根据以下内容写一段科普解说。
要求：通俗易懂，有趣味性，300字以内。

内容：${ctx}`,

  recommend: (ctx) => `你是一个生活达人，请根据以下内容给出相关推荐。
要求：推荐3-5个相关项目，简要说明推荐理由。

内容：${ctx}`,

  travel: (ctx) => `你是一个旅行攻略作者，请根据以下内容生成旅行攻略要点。
要求：实用、简洁、包含必去/必吃项目。

内容：${ctx}`,
}

async function generateContent(context: string, style: Style = 'xiaohongshu'): Promise<string> {
  const client = new Anthropic()
  const response = await client.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1024,
    messages: [{ role: 'user', content: TEMPLATES[style](context) }],
  })
  return response.content[0].text
}
```

## 优化方向

- **风格热切换**：比赛时根据关键词秒换模板
- **内容增强**：结合 Demo 3 的摘要结果，生成更丰富的内容
- **多模态输入**：直接传图片让模型"看图写文"，不依赖上游 Demo
