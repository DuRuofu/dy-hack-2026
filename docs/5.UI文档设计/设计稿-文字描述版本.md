# 设计稿（文字描述版）

> **搭子智能穿搭网页应用** UI 模板（**简约白底**）。支持与外部主流穿搭/购物/内容平台兼容联动，可通过 API 对接实现跨平台数据互通、穿搭信息同步与单品溯源跳转，适配多端生态场景。与《页面结构.md》路由一致；核心 MVP 可无登录、全员共享衣橱。下文「个人中心」用于**职业定制与偏好**（可落地为独立路由或首启一次性选择浮层，与产品叙事一致）。

## 二、各屏关键布局（文字稿）

### 首页 `/`

- 顶区：产品标题或简短说明（可选）；**职业定制**入口（Chip 或「我的穿搭身份」）→ 与个人中心或首启选择联动。
- 主体：**场景卡片列表**（衣架/单品简笔图标 + 场景名 + 一句描述，数据来自 `GET /api/scenes`）。卡片为**白底 + 细线框**（`#EBEBEB`），选中态用**加粗边框或浅灰衬底**（`#F5F5F5`），不用彩色大面铺底。
- 辅助：**「自由搭配」** 按钮/卡片，跳 `/mix-match`（图标建议：拼接衣架 + 星星，表示自选与点评）。
- 底部：**TabBar**（首页 / 衣橱 / 上传 / 推荐 / 淘宝 / 视频；若含个人中心则增加「我的」）。TabBar 区域建议白底 + 顶部分割线，图标默认灰、选中深灰/黑。

### 衣橱 `/wardrobe`（滑动卡片 · 核心体验）

- 顶区：**筛选**（品类、季节、风格等 Chip 或下拉），变更后仍在本页；筛选条底部分割线 `#E8E8E8`，Chip **未选中**白底灰字，**选中**浅灰底 `#F5F5F5` + 深色字 + **1px `#E0E0E0` 边框**（全站不用蓝）。
- 主体：**滑动卡片带**（新鲜感主交互），替代传统密铺网格为主展示形态：
  - **横向滑动**：中间一张为**主卡片**（大图 + 品类/颜色等标签），左右两侧露出相邻卡的边缘（peek，约 **8%～12%** 屏宽），暗示可滑。
  - **操作**：触摸 **横滑**切换；桌面端支持 **拖拽**、**左右方向键**、可选两侧 **箭头**，统一用 `scroll-snap` **居中吸附**当前卡。
  - **动效**：切换时 **200～280ms** 缓动；轻微 **scale(1)→(0.98)** 仅在拖拽中，释放后回弹，避免眩晕。
  - **指示**：底部 **圆点分页**或 **「3 / 12」** 文案，与当前 `clothes.id` 或列表索引同步；筛选后列表变化则指示重置。
  - **进入详情**：**点按当前主卡片**（或卡片上「查看」文字链）→ **`/wardrobe/:id`**；滑动与点击需做手势阈值区分（短滑不误触跳转）。
- **无删除按钮**（与 MVP 约束一致）。
- 可选：**「自由搭配」** 入口 → `/mix-match`（顶栏或卡片带下方）。

### 衣物详情 `/wardrobe/:id`

- 大图 + 名称/品类/颜色/风格/季节等；可编辑字段与 **保存**（`PUT`）。
- 返回 → 衣橱或上一页；返回衣橱时尽量**恢复上次滑动位置**（列表索引记入 Pinia / session），减少割裂感。

### 拍照上传 `/upload`

- **拍照 / 选图** 主按钮；预览区白底。
- 识别结果表单（可修正）+ **保存到衣橱**。
- 无删除、无账号区。

### 视频解析 `/video`

- 输入框：视频链接；**解析** 按钮。
- 结果：**可勾选** 的识别单品列表；**保存选中到衣橱**。

### 淘宝导入 `/taobao`

- 输入框：商品链接；**解析**。
- 预览主图与识别字段；**确认保存到衣橱**。

### 搭配推荐 `/outfit`

- 顶部可显示当前场景名；**换场景** → `/`。
- 主体：**2～3 套** `OutfitCard`（每套含多单品缩略图、分数、理由摘要），卡片样式与首页一致：**白底 + 细边框**。
- 点击整套卡片 → **`/outfit/:id`**。

### 搭配方案详情 `/outfit/:id`

- 横向或纵向排列本套 **上衣 / 下装 / 鞋** 大图；完整理由与评分。
- 点某一单品 → **`/wardrobe/:id`**。

### 自由搭配 `/mix-match`

- 上半：从衣橱选品——可用 **横向滑动条/缩略卡** 与列表页视觉语言统一，或保留紧凑网格（实现择一）；下方 **让 AI 点评** 区同前。
- **让 AI 点评**：同页下方展示分数、`pros` / `cons` / `suggestion`。

### 个人中心 `/profile`（或与首启职业选择合并）

- **职业身份**：互联网 / 教师 / 公职人员 / 设计师 / 博主 / 摄影师 / 大学生 / 研究生等（分组映射到三大类：职场白领 / 创意自由职业 / 学生）。
- **穿搭偏好**：可选标签（如通勤、校园、约会）仅影响文案与默认场景排序，不改变后端共享数据模型。
- **保存**：写入本地（如 `localStorage`）或 Pinia 持久化，全站按钮样式随职业切换（**灰阶与线框层次**变化，见第四章）。

## 三、设计规范

整体方向：**大面积白底、少色、靠排版与留白建立层次**；组件意象仍围绕**衣架、衣物、鞋履、搭配**，图标与摄影图承担色彩，界面本身保持中性。

| 元素 | 值 |
|------|---|
| 页面背景 | `#FFFFFF`；可选次级分区 `#FAFAFA` |
| 表面/卡片 | `#FFFFFF`，边框 `#EBEBEB`（或 `#E8E8E8`） |
| 分割线 | `#F0F0F0` / `#E8E8E8` |
| 主文字 | `#141414`～`#1A1A1A` |
| 次级文字 | `#737373`～`#8C8C8C` |
| 占位/禁用 | `#BDBDBD`、`#E0E0E0` |
| 主操作色 | `#141414`（实心按钮底 + 白字） |
| 弱强调 | 仅用 **透明度** 或 **#404040**～`#6B6B6B`**，不使用蓝/青系品牌色** |
| 字体 | `Inter, "PingFang SC", "Microsoft YaHei", sans-serif` |
| 圆角 | 主按钮 **16px**；次要按钮 **12px**；卡片 **16px**（衣橱滑动主卡可 **20px** 略放大仪式感） |
| 间距 | 基础单位 **4px**；常用 **8px / 16px / 24px / 32px** |
| 阴影 | 极轻：`0 1px 2px rgba(0,0,0,0.06)`；悬浮卡：`0 8px 24px rgba(0,0,0,0.08)`，避免重阴影 |

**补充约定（穿搭主题 + 白底统一感）**

- **禁止**：整页彩色渐变、天蓝/钢蓝主色块、彩色 Tab 条（与旧版划界）。
- 图标：线性、**1.5px** 左右线宽；衣类符号为主，职业仅小徽标。
- 成功态：可用 **极小面积** 绿色对勾图标，不整按钮改绿。
- **衣橱滑动带**：背景保持白或极浅灰，主卡片与白底页之间靠 **阴影 + 边框** 托起层次。

## 四、职业定制化按钮设计

> 三大类职业共用**同一套黑白灰体系**，仅通过**深浅、线框粗细、填充/描边、图标**区分气质；**不出现蓝色**。文案与图标仍须与**穿搭场景**强相关。

**通用尺寸约定**

- 主按钮高度 **48px**（移动端可 **44px**），左右内边距 **24px**；图标区 **20×20px**，与文字间距 **8px**。
- 次按钮高度 **40px**，圆角 **12px**。
- 全宽主按钮在表单页（解析、上传）贴边留白 **16px**。

**交互（全站一致）**

- **Hover**（键鼠）：主按钮背景 `#2C2C2C`；次按钮边框 `#BDBDBD` → `#9E9E9E`；图标与文字 **translateY(-1px)**。
- **Active / 点击**：**scale(0.98)**，**150ms** `ease-out`。
- **禁用**：背景 `#F5F5F5`，文字 `#BDBDBD`，边框 `#E8E8E8`，无位移。

## 五、页面按钮适配说明

### 1. 首页 `/`

- **核心按钮**：主按钮遵循当前职业（白领实心黑灰 / 创意描边 / 学生浅灰底）；「自由搭配」为**次按钮**（白底 + 灰边框，12px 圆角）。
- **随职业切换**：仅换**填充/描边/字重**，不换色相。
- **场景卡片**：白底、细边框；选中 **`#F5F5F5` 衬底 + `#141414` 边框 1px**，不用彩色描边。

### 2. 视频解析页 `/video`

- **核心按钮**：「解析」「保存」以 **`#141414` 实心** 或 **白底主描边**（创意预设）二选一映射职业；进度条可用 **`#141414` 半透明** 或深灰，不用蓝条。
- **列表区**：白底卡片分行，**1px** 分割；品类用小线性图标（鞋/上衣）灰阶着色。

### 3. 衣橱页 `/wardrobe`

- **滑动卡片**：主卡 **白底 + `#EBEBEB` 边框 + 轻阴影**；peek 邻卡略降低透明度 **0.92** 或略缩 **scale(0.96)**（实现择一，避免双标准）。
- **筛选 Chip**：选中浅灰底，**无彩色**；与滑动带之间 **16px** 间距。
- **核心按钮**：「自由搭配」等为次按钮灰框样式。
- **随职业切换**：白领可在主卡角上加**极小**「通勤感」线框标签（灰字）；创意可略增大圆角视觉（仍 **16～20px**）；学生可在分页点用**稍大触摸热区**。
- **新鲜感要点**：首进衣橱可 **自动轻晃第一张卡 1 次**（translateX 4px 回弹，需 `prefers-reduced-motion` 降级为静止）提示可滑。

### 4. 搭配推荐页 `/outfit`

- **核心按钮**：「换场景」为**文本链** `#141414` 下划线可选；「生成 / 刷新」随职业；「查看详情」40px 次按钮。
- **OutfitCard**：白底、细边框；分数徽章 **灰底黑字** 或 **黑底白字**（二选一全局统一）。

### 5. 个人中心 `/profile`（或首启职业浮层）

- **核心按钮**：「保存职业与偏好」默认 **`#141414` 实心**（全职业统一亦可）；选项卡片选中：**`#FAFAFA` 底 + `#E0E0E0` 边框**。
- **顶部**：白底或极浅灰 **`#FAFAFA`** + **衣架线稿**单色灰，**无渐变蓝头图**。
