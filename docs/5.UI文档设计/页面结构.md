# 页面结构

> 与《需求和技术拆分对接》《后端架构》《前端架构》对齐：**无登录、全员共享衣橱、前端不提供删除**（后端无 DELETE）。与《设计稿-文字描述版本.md》对齐：**搭子智能穿搭**、**简约白底** UI、**衣橱横向滑动卡片**主展示、**职业定制化**（仅前端状态，见下）。下列为页面清单、主要按钮/可操作元素，以及点击后的**路由跳转**（`query` 表示查询参数，实现方式可与 Pinia 状态二选一）。

**职业与偏好（无后端接口）：** 用户职业大类（职场白领 / 创意自由职业 / 学生）及穿搭偏好标签仅存 **Pinia + `localStorage`（或等同）**，用于切换全站主按钮样式与部分文案；**不改变**共享衣橱与推荐 API 的数据模型。可选 **首启一次性浮层** 选择职业，与 **`/profile`** 二选一或并存（浮层完成后仍可进「我的」修改）。

## 一、路由与页面对照

| 页面名称 | 路由 | 主要数据接口 | 说明 |
|---------|------|-------------|------|
| 首页（场景选择） | `/` | `GET /api/scenes`；进入推荐结果前会调 `POST /api/outfit/recommend` | 展示预置场景，发起「按场景推荐」；顶区「职业定制」入口 |
| 衣橱列表 | `/wardrobe` | `GET /api/wardrobe`（支持 `?category=&style=&season=`） | **横向滑动卡片带**（snap 居中、peek 邻卡）+ 顶区筛选；点击进入详情；无删除入口 |
| 衣物详情 | `/wardrobe/:id` | `GET /api/wardrobe/:id`；保存编辑 `PUT /api/wardrobe/:id` | 从衣橱或搭配流中点单品进入 |
| 拍照上传 | `/upload` | `POST /api/recognize`；入库 `POST /api/wardrobe` | 拍照/选图 → 识别 → 修正 → 保存 |
| 视频解析 | `/video` | `POST /api/video/parse`；批量入库 `POST /api/video/save` | 抖音链接 → 预览列表 → 勾选保存 |
| 淘宝导入 | `/taobao` | `POST /api/taobao/parse`；入库 `POST /api/wardrobe` | 商品链接 → 预览 → 确认保存 |
| 搭配推荐（结果列表） | `/outfit` | `POST /api/outfit/recommend`（常由首页带场景触发） | 展示 2～3 套 AI 方案卡片 |
| 搭配方案详情 | `/outfit/:id` | 一般复用已进入推荐页时的内存/Pinia 数据；单品信息用 `GET /api/wardrobe/:id` | `:id` 可为方案序号或与后端方案标识一致（MVP 可用列表下标） |
| 自由搭配 + AI 评价 | `/mix-match` | `GET /api/wardrobe`；`POST /api/outfit/evaluate` | 自选上/下/鞋 → 同页展示评分与建议 |
| 个人中心（职业与偏好） | `/profile` | **无**（仅读写本地/前端 store） | 职业身份细项、穿搭偏好标签、保存后驱动全站按钮主题；可与首启浮层合并实现 |

**说明：** 需求文档「页面-接口总览」中将搭配详情写在 `/outfit/:id` 且示例接口为 `GET /api/wardrobe/:id`，容易与「单品详情」混淆。本 UI 约定：**单品详情统一为 `/wardrobe/:id`**（与 REST 一致）；**某一套推荐方案的展开页为 `/outfit/:id`**。

## 二、全局导航（建议）

与整体架构图中的模块一致，可采用 **底部 TabBar**（**6～7 项**：含「我的」时为 7 项；小屏可做横向滚动或「更多」收纳，此处按功能完整列出）：

| Tab 标签（建议文案） | 目标路由 |
|---------------------|----------|
| 首页 | `/` |
| 衣橱 | `/wardrobe` |
| 上传 | `/upload` |
| 推荐 | `/outfit` |
| 淘宝 | `/taobao` |
| 视频 | `/video` |
| 我的（可选，与《设计稿》一致） | `/profile` |

**自由搭配**（`/mix-match`）建议放在：**首页显著入口按钮**，以及 **衣橱页顶栏/空状态辅助入口**（见下表）。

**与职业定制联动：** TabBar 不直接切换职业；进入 **`/profile`** 或完成首启浮层后，各页主按钮样式按设计稿「职业三大类」映射（**灰阶与线框层次**，详见《设计稿-文字描述版本》第四、五章）。

## 三、分页面：按钮 / 操作与跳转

### 1. 首页 `/`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 「职业定制」「我的穿搭身份」Chip / 文字链 | 点击 | **`/profile`**；若采用首启浮层且未实现独立页，可改为 **弹出职业选择**（仍建议路由可深链到 `/profile` 便于分享书签） |
| 场景卡片（如「通勤上班」） | 点击 | 调用 `POST /api/outfit/recommend` 后进入 **`/outfit`**（建议带 `query`：`scene` 或 `sceneId`，便于返回与刷新） |
| 「自由搭配」入口 | 点击 | **`/mix-match`** |
| 底部 Tab「衣橱 / 上传 / 推荐 / 淘宝 / 视频 / （我的）」 | 点击 | 对应路由 **`/wardrobe`**、**`/upload`**、**`/outfit`**、**`/taobao`**、**`/video`**、**`/profile`**（启用「我的」时） |

---

### 2. 衣橱列表 `/wardrobe`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 品类 / 季节 / 风格等筛选 | 点击 | **停留本页**，刷新列表查询参数；**重置滑动索引**为第 1 张（或 0） |
| **滑动卡片带**（横向） | 横滑 / 拖拽 / 方向键 / 可选左右箭头 | **停留本页**，`scroll-snap` 切换当前主卡；更新底部分页指示（圆点或 `当前/总数`） |
| **当前主卡片**（点按，非拖拽手势） | 点击 | **`/wardrobe/:id`**（`:id` 为 `clothes.id`）；建议 **Pinia / session 记录当前索引**，从详情返回时恢复滑动位置 |
| 分页指示（圆点或数字） | 点击（若实现） | **停留本页**，滚动到对应卡片 |
| 「自由搭配」辅助入口（顶栏或卡片带下方） | 点击 | **`/mix-match`** |
| 「去选场景推荐」类入口（可选） | 点击 | **`/`** |
| 底部 Tab | 点击 | 同首页表 |

---

### 3. 衣物详情 `/wardrobe/:id`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 「保存 / 完成编辑」（元信息表单） | 点击 | **停留本页**，`PUT /api/wardrobe/:id` 成功后更新展示 |
| 返回 | 点击 | **`router.back()`** 或 **`/wardrobe`**（产品可二选一固定） |

### 4. 拍照上传 `/upload`

|------|------|------------|

| 「识别」（或选图后自动识别） | 点击 | **停留本页**，`POST /api/recognize` |
| 「保存到衣橱」 | 点击 | `POST /api/wardrobe` 成功后可选：**停留**并清空表单，或 **`/wardrobe`** 查看新衣 |
| 「取消 / 返回」 | 点击 | **`router.back()`** 或 Tab 切换 |
| 底部 Tab | 点击 | 同首页表 |

### 5. 视频解析 `/video`

| 「保存选中到衣橱」 | 点击 | `POST /api/video/save` 成功后可选：**停留**或 **`/wardrobe`** |

### 7. 搭配推荐 `/outfit`

### 10. 个人中心 `/profile`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 职业大类单选（职场白领 / 创意自由职业 / 学生） | 点击 | **停留本页**，更新 Pinia；细分子职业（互联网、教师、设计师等）映射到三大类之一 |
| 穿搭偏好标签（如通勤、校园、约会）（可选） | 点击 | **停留本页**，仅影响默认场景排序或首页文案，**不调后端** |
| 「保存职业与偏好」 | 点击 | **写入 `localStorage` + Pinia`，**停留本页**或 **`router.back()`**；全站主按钮/图标按《设计稿》第四章切换（**黑白灰体系**） |
| 首启职业浮层（若与路由并存） | 确认 | 同上，关闭浮层后可进入 **`/`** |
| 返回 | 点击 | **`router.back()`** 或 **`/`** |
| 底部 Tab「我的」 | 点击 | **停留 `/profile`**（当前页） |

## 四、用户主路径简图

```
首页 / →（选场景）→ 搭配推荐 /outfit →（点某套）→ 搭配方案详情 /outfit/:id
                                                      →（点单品）→ 衣物详情 /wardrobe/:id

衣橱 /wardrobe →（横滑选卡）→（点当前主卡）→ 衣物详情 /wardrobe/:id

上传 /upload、视频 /video、淘宝 /taobao →（保存后）→ 可选跳转 衣橱 /wardrobe

首页 / 或 衣橱 →（自由搭配）→ /mix-match（AI 点评不强制跳转新页）

首页 / →（职业定制）→ /profile（保存后返回，全站按钮主题更新）
```
