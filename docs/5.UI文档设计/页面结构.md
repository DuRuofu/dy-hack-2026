# 页面结构

> 与《需求和技术拆分对接》《后端架构》《前端架构》对齐：**无登录、全员共享衣橱、前端不提供删除**（后端无 DELETE）。下列为页面清单、主要按钮/可操作元素，以及点击后的**路由跳转**（`query` 表示查询参数，实现方式可与 Pinia 状态二选一）。

---

## 一、路由与页面对照

| 页面名称 | 路由 | 主要数据接口 | 说明 |
|---------|------|-------------|------|
| 首页（场景选择） | `/` | `GET /api/scenes`；进入推荐结果前会调 `POST /api/outfit/recommend` | 展示预置场景，发起「按场景推荐」 |
| 衣橱列表 | `/wardrobe` | `GET /api/wardrobe`（支持 `?category=&style=&season=`） | 网格列表 + 筛选，无删除入口 |
| 衣物详情 | `/wardrobe/:id` | `GET /api/wardrobe/:id`；保存编辑 `PUT /api/wardrobe/:id` | 从衣橱或搭配流中点单品进入 |
| 拍照上传 | `/upload` | `POST /api/recognize`；入库 `POST /api/wardrobe` | 拍照/选图 → 识别 → 修正 → 保存 |
| 视频解析 | `/video` | `POST /api/video/parse`；批量入库 `POST /api/video/save` | 抖音链接 → 预览列表 → 勾选保存 |
| 淘宝导入 | `/taobao` | `POST /api/taobao/parse`；入库 `POST /api/wardrobe` | 商品链接 → 预览 → 确认保存 |
| 搭配推荐（结果列表） | `/outfit` | `POST /api/outfit/recommend`（常由首页带场景触发） | 展示 2～3 套 AI 方案卡片 |
| 搭配方案详情 | `/outfit/:id` | 一般复用已进入推荐页时的内存/Pinia 数据；单品信息用 `GET /api/wardrobe/:id` | `:id` 可为方案序号或与后端方案标识一致（MVP 可用列表下标） |
| 自由搭配 + AI 评价 | `/mix-match` | `GET /api/wardrobe`；`POST /api/outfit/evaluate` | 自选上/下/鞋 → 同页展示评分与建议 |

**说明：** 需求文档「页面-接口总览」中将搭配详情写在 `/outfit/:id` 且示例接口为 `GET /api/wardrobe/:id`，容易与「单品详情」混淆。本 UI 约定：**单品详情统一为 `/wardrobe/:id`**（与 REST 一致）；**某一套推荐方案的展开页为 `/outfit/:id`**。

## 二、全局导航（建议）

与整体架构图中的模块一致，可采用 **底部 TabBar**（6 项，小屏可做滚动或「更多」收纳，此处按功能完整列出）：

| Tab 标签（建议文案） | 目标路由 |
|---------------------|----------|
| 首页 | `/` |
| 衣橱 | `/wardrobe` |
| 上传 | `/upload` |
| 推荐 | `/outfit` |
| 淘宝 | `/taobao` |
| 视频 | `/video` |

**自由搭配**（`/mix-match`）建议放在：**首页显著入口按钮**，以及 **衣橱页顶栏/空状态辅助入口**（见下表）。

---

## 三、分页面：按钮 / 操作与跳转

### 1. 首页 `/`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 场景卡片（如「通勤上班」） | 点击 | 调用 `POST /api/outfit/recommend` 后进入 **`/outfit`**（建议带 `query`：`scene` 或 `sceneId`，便于返回与刷新） |
| 「自由搭配」入口 | 点击 | **`/mix-match`** |
| 底部 Tab「衣橱 / 上传 / 推荐 / 淘宝 / 视频」 | 点击 | 对应路由 **`/wardrobe`**、**`/upload`**、**`/outfit`**、**`/taobao`**、**`/video`** |

---

### 2. 衣橱列表 `/wardrobe`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 品类 / 季节 / 风格等筛选 | 点击 | **停留本页**，刷新列表查询参数 |
| 单件衣物卡片 | 点击 | **`/wardrobe/:id`**（`:id` 为 `clothes.id`） |
| 「自由搭配」辅助入口（顶栏或 FAB） | 点击 | **`/mix-match`** |
| 「去选场景推荐」类入口（可选） | 点击 | **`/`** |
| 底部 Tab | 点击 | 同首页表 |

---

### 3. 衣物详情 `/wardrobe/:id`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 「保存 / 完成编辑」（元信息表单） | 点击 | **停留本页**，`PUT /api/wardrobe/:id` 成功后更新展示 |
| 返回 | 点击 | **`router.back()`** 或 **`/wardrobe`**（产品可二选一固定） |

---

### 4. 拍照上传 `/upload`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 「拍照」/「从相册选择」 | 点击 | 唤起系统相机或文件选择，**不离开路由** |
| 「识别」（或选图后自动识别） | 点击 | **停留本页**，`POST /api/recognize` |
| 「保存到衣橱」 | 点击 | `POST /api/wardrobe` 成功后可选：**停留**并清空表单，或 **`/wardrobe`** 查看新衣 |
| 「取消 / 返回」 | 点击 | **`router.back()`** 或 Tab 切换 |
| 底部 Tab | 点击 | 同首页表 |

---

### 5. 视频解析 `/video`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 「解析视频」 | 点击 | **停留本页**，`POST /api/video/parse`，展示可勾选列表 |
| 列表项勾选 | 点击 | **停留本页** |
| 「保存选中到衣橱」 | 点击 | `POST /api/video/save` 成功后可选：**停留**或 **`/wardrobe`** |
| 底部 Tab | 点击 | 同首页表 |

---

### 6. 淘宝导入 `/taobao`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 「解析商品」 | 点击 | **停留本页**，`POST /api/taobao/parse` |
| 「确认保存到衣橱」 | 点击 | `POST /api/wardrobe` 成功后可选：**停留**或 **`/wardrobe`** |
| 底部 Tab | 点击 | 同首页表 |

---

### 7. 搭配推荐 `/outfit`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 某一套方案卡片（整套） | 点击 | **`/outfit/:id`**（`:id` 对应本页列表中的方案下标或方案 id） |
| 「换场景 / 重新选场景」 | 点击 | **`/`** |
| 「再推荐一次」（可选） | 点击 | **停留本页**，再次 `POST /api/outfit/recommend` |
| 底部 Tab | 点击 | 同首页表 |

---

### 8. 搭配方案详情 `/outfit/:id`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 方案内某一单品图（上衣/下装/鞋等） | 点击 | **`/wardrobe/:id`**（衣物 id，调 `GET /api/wardrobe/:id`） |
| 返回 | 点击 | **`/outfit`** 或 **`router.back()`** |

---

### 9. 自由搭配 `/mix-match`

| 元素 | 行为 | 跳转或结果 |
|------|------|------------|
| 衣橱网格中点选单品（按角色：上衣 / 下装 / 鞋） | 点击 | **停留本页**，更新已选组合 |
| 「让 AI 点评」 | 点击 | **停留本页**，`POST /api/outfit/evaluate`，在下方或同页展示分数与文案 |
| 已选单品缩略图（若可点） | 点击 | 可选 **`/wardrobe/:id`** |
| 返回 | 点击 | **`/`** 或 **`/wardrobe`** 或 **`router.back()`**（与入口统一即可） |
| 底部 Tab | 点击 | 同首页表 |

---

## 四、用户主路径简图

```
首页 / →（选场景）→ 搭配推荐 /outfit →（点某套）→ 搭配方案详情 /outfit/:id
                                                      →（点单品）→ 衣物详情 /wardrobe/:id

衣橱 /wardrobe →（点单品）→ 衣物详情 /wardrobe/:id

上传 /upload、视频 /video、淘宝 /taobao →（保存后）→ 可选跳转 衣橱 /wardrobe

首页 / 或 衣橱 →（自由搭配）→ /mix-match（AI 点评不强制跳转新页）
```
