## 二、数据库设计（PostgreSQL）

> **无用户体系，全员共享数据。** 这是 MVP 演示项目，不做用户认证和数据隔离。所有衣物数据对所有访客可见。
> 为防止恶意操作，前端不提供删除按钮，后端不暴露 DELETE 接口。
> 衣物图片存储在阿里云 OSS，数据库只存 OSS URL（元信息）。

```sql
-- 衣物表
CREATE TABLE clothes (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(100) NOT NULL,      -- 名称
  category    VARCHAR(20) NOT NULL,       -- 品类: 上衣/裤子/裙子/鞋/包/配饰
  color       VARCHAR(20),                -- 主色
  style       VARCHAR(20),                -- 风格: 休闲/通勤/运动/甜美
  season      VARCHAR(10),                -- 季节: 春/夏/秋/冬/四季
  oss_url     TEXT,                       -- 阿里云 OSS 图片地址
  source      VARCHAR(20) DEFAULT 'upload', -- 来源: upload/video/taobao
  taobao_url  TEXT,                       -- 淘宝原始链接（来源为 taobao 时）
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 搭配方案表
CREATE TABLE outfits (
  id          SERIAL PRIMARY KEY,
  scene       VARCHAR(50),                -- 场景: 通勤/约会/运动
  items       JSONB NOT NULL,             -- [{"clothe_id": 1, "role": "top"}, ...]
  score       REAL,                       -- AI 评分
  reason      TEXT,                       -- 推荐理由
  is_custom   BOOLEAN DEFAULT FALSE,      -- FALSE=AI推荐 TRUE=用户自选
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 场景表（预置数据）
CREATE TABLE scenes (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(50) NOT NULL,       -- 通勤上班
  icon        VARCHAR(50),                -- 图标名
  description TEXT                        -- "适合商务休闲风格"
);
```

### 存储策略

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| 衣物元信息（品类/颜色/风格） | PostgreSQL | 结构化数据，支持筛选查询 |
| 衣物图片 | 阿里云 OSS | 只存 URL 在数据库，OSS 提供 CDN 加速访问 |
| 搭配方案 | PostgreSQL + Redis 缓存 | 热门场景的推荐结果用 Redis 缓存，减少 AI 调用 |
| 临时视频帧 | 后端本地 `/tmp` | 处理完即删除，不持久化 |

## 三、功能逐个拆解

### 功能 1：抖音视频 → 提取穿搭单品（核心，不可取消）

> 用户粘贴抖音视频链接，系统下载视频、提取关键帧，AI 识别每帧中的服装单品，存入衣橱。

**用户流程：**

```
用户操作                          系统响应
─────────                        ─────────
1. 粘贴视频链接
        ──POST /api/video/parse──▶  2. 下载视频 (yt-dlp)
                                    3. 提取关键帧 (ffmpeg)
                                    4. 每帧送 AI 识别
                                    5. 返回识别出的单品列表
        ◀── JSON 单品列表 ────────
6. 预览确认，勾选要保存的
        ──POST /api/wardrobe───▶    7. 写入 clothes 表
```

**流程图：**

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 用户粘贴  │────▶│ 后端下载   │────▶│ 提取关键帧 │────▶│ AI 识别   │
│ 视频链接  │     │ 视频文件   │     │ (ffmpeg)  │     │ 每帧单品   │
└─────────┘     └──────────┘     └──────────┘     └────┬─────┘
                                                       │
                                                       ▼
                                              ┌──────────────┐
                                              │ 返回单品列表   │
                                              │ 品类/颜色/风格 │
                                              └──────┬───────┘
                                                     │ 用户确认
                                                     ▼
                                              ┌──────────────┐
                                              │ 写入衣橱数据库  │
                                              └──────────────┘
```

**接口设计：**

| 方法 | 路径 | 入参 | 出参 | 说明 |
|------|------|------|------|------|
| POST | /api/video/parse | `{ url: string }` | `{ items: [{ category, color, style, season, image_base64 }] }` | 解析视频，返回识别单品 |
| POST | /api/video/save | `{ items: [{ image_base64, category, color, style, season }] }` | `{ ids: number[] }` | 图片上传 OSS + 写入 DB |

**技术细节：**
- `execa("yt-dlp", [url, "-o", tempPath])` 下载视频
- `execa("ffmpeg", ["-i", videoPath, "-vf", "fps=1", frameDir])` 每秒取 1 帧
- 每帧 base64 编码后调 DashScope 多模态 API 识别
- Prompt 要求返回 JSON: `{ category, color, style, season }`
- 保存时将 base64 图片上传至阿里云 OSS，数据库只存返回的 oss_url

### 功能 2：拍照上传 → AI 识别衣物

> 用户拍照或从相册选图上传，AI 自动识别衣物信息，加入衣橱。

**用户流程：**

```
用户操作                          系统响应
─────────                        ─────────
1. 点击拍照/选图按钮
2. 选择或拍摄照片
3. 预览确认
        ──POST /api/recognize──▶    4. AI 多模态识别
        ◀── { category,color,     ── 5. 返回识别结果
             style,season }──
6. 用户可手动修正信息
        ──POST /api/wardrobe───▶    7. 图片上传 OSS + 写入 DB
```

**流程图：**

```
┌─────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│ 拍照/选图 │────▶│ 预览确认   │────▶│ 上传到后端 │────▶│ AI 识别   │
└─────────┘     └──────────┘     └──────────┘     └────┬─────┘
                                                       │
                           ┌───────────────────────────┘
                           ▼
                    ┌──────────────┐     ┌──────────┐
                    │ 返回识别结果   │────▶│ 用户修正   │
                    │ 品类/颜色/风格 │     │ (可选)    │
                    └──────────────┘     └────┬─────┘
                                              │
                                              ▼
                                       ┌──────────────┐
                                       │ 保存到衣橱    │
                                       └──────────────┘
```

**接口设计：**

| 方法 | 路径 | 入参 | 出参 | 说明 |
|------|------|------|------|------|
| POST | /api/recognize | `multipart/form-data: file` | `{ category, color, style, season, confidence }` | AI 识别衣物 |
| POST | /api/wardrobe | `{ category, color, style, season, oss_url }` | `{ id: number }` | 添加到衣橱（图片存 OSS） |

**技术细节：**
- 前端用 `<input type="file" accept="image/*" capture="environment">` 支持拍照
- 后端接收文件后先上传到阿里云 OSS，拿到 oss_url
- DashScope API 的 `image_url` 字段传入 OSS 地址
- 数据库只存 oss_url，不存图片二进制

**接口设计：**

| 方法 | 路径 | 入参 | 出参 | 说明 |

## 六、关键技术点

> 本项目的三个核心技术环节，需要重点验证和攻克。

### 关键技术点 1：抖音视频 → 衣物单品提取

```
视频链接 → yt-dlp 下载 → ffmpeg 按秒抽帧 → DashScope 逐帧识别 → 结构化衣物数据
```

**技术链路：**
1. `yt-dlp` 解析抖音链接，下载视频文件到 `/tmp`
2. `ffmpeg` 按 1fps 抽取关键帧，输出为图片序列
3. 每帧图片 base64 编码，送 DashScope 多模态 API
4. AI Prompt 指令要求返回结构化 JSON：`{ category, color, style, season }`
5. 去重（同一衣物在多帧中出现），汇总输出单品列表
6. 用户确认后，图片上传 OSS，元信息写入 PostgreSQL

**技术难点：**
- 一个视频可能有 30+ 帧，需并发调用 AI 控制耗时
- 同一件衣服在不同帧中重复出现，需要去重策略（按品类+颜色聚类）
- 抖音链接可能有防盗链，yt-dlp 需要配置 cookies 或代理

---

### 关键技术点 2：淘宝链接 → 衣物图片及信息提取

```
淘宝链接 → 请求商品页 → cheerio 解析 → 提取主图+标题 → AI 结构化 → 入库
```

**技术链路：**
1. 后端接收淘宝商品 URL
2. 用 `cheerio` 解析商品页 HTML：
   - 提取 `<meta property="og:image">` 或主图 `<img>` 的 `src`
   - 提取 `<title>` 作为衣物名称
3. 主图上传至阿里云 OSS，获取 oss_url
4. 标题文字送 DashScope，AI 提取品类、颜色、风格、季节
5. 组装完整衣物信息返回前端预览，用户确认后写入 DB

**技术难点：**
- 淘宝有反爬机制，可能需要模拟 User-Agent 或使用 Puppeteer
- 移动端淘宝链接和 PC 端页面结构不同，需兼容
- 部分商品需要登录才能看到详情，MVP 阶段只处理公开页面

---

### 关键技术点 3：穿搭建议生成 + 可视化展示

```
场景 + 衣橱数据 → Prompt 拼装 → DashScope 文本生成 → JSON 结构化 → 前端卡片渲染
```

**技术链路：**
1. 用户选择场景（通勤/约会/运动…）
2. 后端从 PostgreSQL 查询全部衣物，组装成衣物列表文本
3. 拼装 Prompt：场景描述 + 衣物列表 + 输出格式要求
4. 调用 DashScope 文本生成 API
5. AI 返回 JSON：`{ outfits: [{ items, reason, score }] }`
6. 前端拿到数据后：
   - 按"上衣→下装→鞋"顺序排列单品卡片
   - 展示 AI 评分（进度条/星级）
   - 展示搭配理由（气泡文字）
   - 缺件提示（AI 标注缺少的品类）

**Redis 缓存策略：**
- 缓存 key：`outfit:{scene}:{wardrobe_hash}`（场景 + 衣橱指纹）
- TTL：30 分钟
- 命中缓存直接返回，跳过 AI 调用，节省时间和 token
