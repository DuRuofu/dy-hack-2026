# 数据结构设计

> 涵盖数据库表结构、JSON 数据格式、TypeScript 类型定义。
> 所有数据全员共享，无用户体系。

---

## 一、数据库表结构（PostgreSQL）

### 1.1 clothes — 衣物表

```sql
CREATE TABLE clothes (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(100) NOT NULL,       -- 名称，如"白色亚麻衬衫"
  category    VARCHAR(20) NOT NULL,        -- 品类
  color       VARCHAR(20),                 -- 主色
  style       VARCHAR(20),                 -- 风格
  season      VARCHAR(10),                 -- 季节
  oss_url     TEXT,                        -- 阿里云 OSS 图片地址
  source      VARCHAR(20) DEFAULT 'upload',-- 来源
  taobao_url  TEXT,                        -- 淘宝原始链接
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

**枚举值定义：**

| 字段 | 可选值 |
|------|--------|
| category | `top` 上衣, `bottom` 下装, `dress` 连衣裙, `shoes` 鞋, `bag` 包, `accessory` 配饰 |
| color | `black` 黑, `white` 白, `red` 红, `blue` 蓝, `green` 绿, `yellow` 黄, `pink` 粉, `gray` 灰, `brown` 棕, `beige` 米, `multi` 多色 |
| style | `casual` 休闲, `formal` 通勤, `sport` 运动, `sweet` 甜美, `street` 街头, `minimal` 极简 |
| season | `spring` 春, `summer` 夏, `autumn` 秋, `winter` 冬, `all` 四季 |
| source | `upload` 拍照上传, `video` 抖音视频, `taobao` 淘宝链接 |

### 1.2 outfits — 搭配方案表

```sql
CREATE TABLE outfits (
  id          SERIAL PRIMARY KEY,
  scene       VARCHAR(50),                 -- 适用场景
  items       JSONB NOT NULL,              -- 搭配单品组合（见下方结构）
  score       REAL,                        -- AI 评分 0-100
  reason      TEXT,                        -- 搭配理由
  is_custom   BOOLEAN DEFAULT FALSE,       -- false=AI推荐, true=用户自选
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

**items JSONB 结构：**

```json
[
  { "clothe_id": 1,  "role": "top" },
  { "clothe_id": 5,  "role": "bottom" },
  { "clothe_id": 12, "role": "shoes" },
  { "clothe_id": 8,  "role": "accessory" }
]
```

| 字段 | 说明 |
|------|------|
| clothe_id | 关联 clothes 表的 id |
| role | 该单品在搭配中的角色：`top` 上衣, `bottom` 下装, `shoes` 鞋, `bag` 包, `accessory` 配饰 |

### 1.3 scenes — 场景表（预置常量）

```sql
CREATE TABLE scenes (
  id          SERIAL PRIMARY KEY,
  name        VARCHAR(50) NOT NULL,
  icon        VARCHAR(10),                 -- emoji 图标
  description TEXT
);
```

**预置数据：**

| name | icon | description |
|------|------|-------------|

## 二、API 数据格式

### 2.1 衣物对象（Clothing）

所有接口返回衣物时统一使用此结构：

```typescript
interface Clothing {
  id: number;
  name: string;            // "白色亚麻衬衫"
  category: string;        // "top"
  color: string;           // "white"
  style: string;           // "casual"
  season: string;          // "summer"
  oss_url: string;         // "https://xxx.oss-cn-hangzhou.aliyuncs.com/xxx.jpg"
  source: string;          // "upload" | "video" | "taobao"
  taobao_url?: string;     // 淘宝链接（source=taobao 时有值）
  created_at: string;      // ISO 8601 时间戳
}
```

### 2.2 搭配方案对象（OutfitPlan）

```typescript
interface OutfitItem {
  clothe_id: number;
  role: string;            // "top" | "bottom" | "shoes" | "bag" | "accessory"
  // 以下字段由后端 JOIN clothes 表后填充
  name: string;
  color: string;
  oss_url: string;
}

interface OutfitPlan {
  items: OutfitItem[];     // 搭配中的单品（带详情）
  score: number;           // AI 评分 0-100
  reason: string;          // "经典黑白配，干练又不失品味"
  is_custom: boolean;      // 是否用户自选
}
```

### 2.3 AI 识别结果

AI 多模态识别单张图片后返回的结构：

```typescript
interface RecognitionResult {
  category: string;        // "top"
  color: string;           // "white"
  style: string;           // "casual"
  season: string;          // "summer"
  confidence: number;      // 置信度 0-1
  name?: string;           // AI 推断的名称，如"白色亚麻衬衫"
}
```

### 2.4 AI 审美评价

用户自选搭配后 AI 返回的评价结构：

```typescript
interface Evaluation {
  score: number;           // 0-100
  pros: string[];          // ["色彩搭配和谐", "风格统一"]
  cons: string[];          // ["鞋子与上衣风格不搭"]
  suggestion: string;      // "建议将运动鞋换成乐福鞋"
}
```

### 2.5 抖音视频解析结果

```typescript
interface VideoParseResult {
  items: {
    category: string;
    color: string;
    style: string;
    season: string;
    image_base64: string;  // 关键帧图片 base64（临时，保存时上传 OSS）
  }[];
}
```

### 2.6 淘宝链接解析结果

```typescript
interface TaobaoParseResult {
  name: string;            // 商品标题
  category: string;        // AI 从标题推断
  color: string;
  style: string;
  season: string;
  oss_url: string;         // 主图已上传 OSS
  taobao_url: string;      // 原始链接
}
```

## 三、数据关系图

```
┌──────────┐       items JSONB        ┌──────────┐
│  outfits  │ ──────┐                 │  scenes   │
│           │       │                 │           │
│  id       │       │  ┌─────────┐   │  id       │
│  scene    │───────┼─▶│clothes  │   │  name     │
│  items    │       │  │         │   │  icon     │
│  score    │       └─▶│id       │   │  desc     │
│  reason   │          │name     │   └──────────┘
└──────────┘          │category │
                      │color    │
                      │oss_url ─┼──▶ 阿里云 OSS（图片文件）
                      │source   │
                      └─────────┘

clothes ←── 1:N ──→ outfits（通过 items JSONB 中的 clothe_id 关联）
scenes ←── 1:N ──→ outfits（通过 scene 字段关联）
```

## 四、Redis 缓存结构

| Key 格式 | Value | TTL | 说明 |
|---------|-------|-----|------|
| `outfit:{scene}:{wardrobe_hash}` | `OutfitPlan[]` JSON | 30min | 推荐结果缓存 |

`wardrobe_hash` = 对当前衣橱所有衣物 id 排序后做 MD5，衣橱变化后缓存自动失效。
