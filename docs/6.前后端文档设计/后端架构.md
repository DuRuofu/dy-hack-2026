# 后端架构

## 技术选型

- Node.js + TypeScript
- NestJS 框架
- PostgreSQL + Drizzle ORM
- Redis 缓存
- 阿里云 OSS（图片存储）
- AI 多模态：通义千问 (DashScope) + 豆包（可切换）
- yt-dlp + ffmpeg（视频处理）
- cheerio（淘宝页面解析）

## API 清单

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/recognize | 图片识别衣物 |
| GET | /api/wardrobe | 获取衣橱列表 |
| POST | /api/wardrobe | 添加衣物（图片存 OSS） |
| PUT | /api/wardrobe/:id | 编辑衣物元信息 |
| GET | /api/wardrobe/:id | 单品详情 |
| POST | /api/video/parse | 抖音视频解析 |
| POST | /api/video/save | 保存视频提取的衣物 |
| POST | /api/taobao/parse | 淘宝链接解析 |
| GET | /api/scenes | 场景列表 |
| POST | /api/outfit/recommend | 搭配推荐（Redis 缓存） |
| POST | /api/outfit/evaluate | 自选搭配 AI 评价 |

> 注意：无 DELETE 接口，防止恶意清空共享数据。

## 数据库表

| 表 | 说明 |
|----|------|
| clothes | 衣物（品类、颜色、风格、oss_url、来源） |
| outfits | 搭配方案（JSONB items、评分、理由） |
| scenes | 预置场景（通勤/约会/运动） |

## AI 多模型切换设计

通过 `.env` 环境变量控制使用哪个模型，业务代码不感知具体实现。

```
AI_PROVIDER=dashscope    # 或 doubao
DASHSCOPE_API_KEY=xxx
DOUBAO_API_KEY=xxx
```

**统一接口：**

```typescript
// ai.interface.ts
export interface IAiProvider {
  // 图片识别：传入图片 URL，返回结构化衣物信息
  recognizeClothing(imageUrl: string): Promise<ClothingInfo>;
  // 搭配推荐：传入衣物列表 + 场景，返回搭配方案
  recommendOutfit(clothes: Clothing[], scene: string): Promise<OutfitPlan[]>;
  // 审美评价：传入选中的衣物，返回评分和建议
  evaluateOutfit(clothes: Clothing[]): Promise<Evaluation>;
}
```

**切换方式：** `ai.module.ts` 根据 `AI_PROVIDER` 环境变量，动态注入对应的 provider。

```typescript
// ai.module.ts
@Module({
  providers: [
    {
      provide: IAiProvider,
      useFactory: (config: ConfigService) => {
        const provider = config.get('AI_PROVIDER');
        if (provider === 'doubao') return new DoubaoProvider(config);
        return new DashScopeProvider(config);  // 默认
      },
      inject: [ConfigService],
    },
  ],
  exports: [IAiProvider],
})
export class AiModule {}
```

各业务模块（recognize、outfit）只注入 `IAiProvider`，不关心底层用的是哪个模型。换模型 = 改 `.env` 一行配置。
